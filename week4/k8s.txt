(kubectl with minikube)

kubectl create deployment name --image=imageName
kubectl get pods 
kubectl delete pod name 
kubectl delete deployment name (if restarts again after deleting a pod)

-details about a deployment 
kubectl describe deployment dep_name 

-details about a pod(s)
kubectl describe pod pod_name 


-to stop a deployment 
kubelet scale deployment name --replicas=0

-delete the deployment
kubectl delete deployment dep_name 

-delete service
kubectl delete service service_name 


--services
-get a cluster ip for a service 

kubectl expose deployment dep_name --port=port_no  --target-port=pods_port_no

-node port 

kubectl expose deployment dep_name --type=NodePort --port=port_no 

-loadbalancer 

kubectl expose deployment my-app --type=LoadBalancer --port=5000 

kubectl expose deployment my-app --type=NodePort --port=80 --target-port=5000
->random generated port -> 80 for node ->5000 for pod


and 

kubectl get services






to roll out an update for an exisiting pod

kubectl set image deployment node-for-k8s test-node-for-k8s=aadarsh660/test-node-for-k8s:2.0
                            deployment name   container name = new container name         
then 
kubectl rollout status deploy node-for-k8s


--delete all (deployment, services)

kubectl delete all --all
